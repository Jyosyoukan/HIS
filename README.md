 《程序设计基础课程设计》 （2025级）

编写一个C语言程序，实现一个适用于小型医院的轻量级的医疗管理系统（HIS） ，能够管理至少100名普通患者 ，30名住院患者，20名医生 ，5个科室，每个科室五个房间，10个床位，共50个三种病房 ，30类药品， 并包含以下功能：
 
登录窗口
【用户名】输入身份证号作为用户名，用于检索用户
【密码】  用于验证身份
【注册】输入姓名，身份证号，密码，可以注册医生，患者，或者是药品管理员
	为了简化，约定药品管理员只有一位，只需要一个统一账号，用户名需要满足一定的条件，包括但不限于长度合法，不允许非法字符等。
	对于不同身份的用户，分别实现下列功能：
 
病人
【挂号】预约挂号 → 选择科室 → 能看到一周内的各个医生 → 选择周几 → 医生详细信息 → 选择医生 → 缴费
		约定显示一周内的预约信息，程序通过数字引索科室，科室可以包含内科，外科，妇产科，儿科，骨科等，不同科室包含一些检查和对应药品。
【当日挂号】患者当日挂号的页面。
	流程包括：选择科室 → 选择医生 → 缴费
【查看已挂号】程序提供显示已经挂的号
【账单】程序提供查看过往记录
【查看自己的信息】程序显示包括姓名，性别等个人信息。
【诊断 / 病情查看】查看自己病情，医生开的药品等。
【住院业务】 程序显示住院床位，住院日期，出院日期等信息。
 
医生
【自己的信息】：程序为医生显示姓名、科室、职称等个人信息。
【查看病人列表 】→ 选择病人 → 写病例，
	规定病例包含：住院开始日期，住院结束日期，给病人开的药品。
【查询病人的详细信息】查询病人姓名，性别，病例等信息，并可以修改。
【诊断/修改病例】医生根据患者病情，为病人开药，如果需要检查，为病人提供对应检查，如果需要住院，为病人提供对应科室的住院床位。
	关于床位，程序规定床位数量有限，每个科室床位数量固定，患者得到床位，空闲床位减少，如果科室没有床位，则医生无法再为患者分配床位，患者出院，床位会自动转为空闲。
 
药品管理人员
【查看药品列表】：对于每个药品，程序会显示药品编号、名称、日期、单价、库存等。
关于药品信息，程序提供处方药非处方药的信息显示，并可以用别名和常用名检索，且部分药品与科室绑定。
【药品数量管理】药品管理人员可以修改药品的数量，以便在药品入库和出库时，准确录入信息。
	约定发药时，实现库存在开药时自动减少
【查看药品库存】程序提供显示药品库存的页面。
